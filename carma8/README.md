## Описание проекта

В проекте реализован функционал, похожий на TikTok / YouTube Shorts / Instagram Reels.

Я постарался предусмотреть сценарии использования, близкие к реальному приложению. Приложение стабильно работает при просмотре видео и поддерживает дозагрузку контента при плохом интернете. Лента является бесконечной — новые видео подгружаются по мере поступления ссылок с backend.

Поддерживаются как мобильный, так и desktop-режимы. В desktop-версии появляются дополнительные элементы управления, если ширина экрана больше высоты.

Реализован спиннер во время загрузки новых видео.

Оптимизация ресурсов достигается за счёт:

* предзагрузки соседних видео,
* удаления уже просмотренных элементов,
* повторного использования ссылок на видео при возврате пользователя.

## Описание решения

Для реализации листания видео использовался `IntersectionObserver`, что позволило снизить нагрузку.

В каждый момент времени отслеживается только один элемент — тот, который находится в области видимости.

В процессе разработки возникли проблемы с нестабильной работой `IntersectionObserver` на мобильных устройствах (особенно при инерционном скролле). В такие моменты обработчик вызывался множество раз.

Проблема усиливалась при работе с DOM, особенно при добавлении или удалении элементов в начало списка. Предположительно это связано с перерасчётом layout.

Для оптимизации:

* использовалась функция `replaceChildren`,
* минимизировались операции с DOM,
* добавлен класс `Scheduler`, который выполняет действия после завершения события `scroll`.

Это позволило стабилизировать работу `IntersectionObserver`.

При работе с концом списка подобных проблем не возникает — добавление и удаление элементов работает стабильно.

Также для ускорения поиска элементы видео, находящиеся в DOM, кэшируются в отдельных структурах (VideoItem).

## Описание сущностей

* **Video (video/video-entity.js)** — сущность данных, полученных с backend
* **VideoItem (video/video-item.js)** — управление и очистка DOM-элементов видео
* **VideoDispenser (video/video-dispenser.js)** — загрузка видео и кэширование
* **VideoController (video/video-controller.js)** — реализация infinite scroll и оптимизация переключения видео; связывает остальные сущности
* **Scheduler (utils/scheduler.js)** — вспомогательный планировщик для стабилизации работы с DOM
* **Mock (utils/mock.js)** — генератор (`function*`) для эмуляции backend
* **PlayList (play-list/play-list.js)** — менеджер `VideoItem`
* **PlayListControls (play-list/play-list-controls.js)** — управление контролами
* **LoaderController (loader/loader-controller.js)** — управление загрузчиком ленты

## Личные комментарии

Можно улучшить работу с DOM за счёт синхронизации операций добавления и удаления элементов — сейчас они выполняются независимо.

Также можно рассмотреть использование Web Components для работы с видеоэлементами.

CSS-часть проработана минимально — использовались только необходимые классы.

Работа с бесконечным списком требует учитывать перерасчёт layout. Возможно, использование `transform` вместо `scroll-snap` даст больше контроля, но потребует реализации собственной логики скролла.

Для сравнения изучал поведение YouTube Shorts — у них DOM-элементы выше текущего видео не удаляются, возможно, из-за нестабильности подобных оптимизаций.
